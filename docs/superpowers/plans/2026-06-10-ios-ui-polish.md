# Aiszr iOS 极简高级 UI 优化计划

## 目标

把 Aiszr 从“功能堆叠型桌面工具”打磨成“iOS 控制中心 + 专业直播中控台”风格的应用：界面克制、层级清晰、动效自然、状态反馈准确，适合直播中长时间盯屏操作。

## 设计原则

1. **少即是高级**
   - 首页只保留直播时高频查看和操作的信息。
   - 配置项进入对应配置页，不在首页展开完整表单。
   - 降低大段说明文字，优先用状态点、短标签、图标和紧凑控件表达。

2. **统一材质和层级**
   - 页面背景、卡片、输入框、按钮、开关统一使用一套 token。
   - 卡片只用于真正独立的信息块，避免卡片套卡片。
   - 阴影和边框要轻，避免安卓化、后台系统化的厚重感。

3. **动效服务状态，不做炫技**
   - 页面切换使用短距离位移 + 淡入，不做大幅滑动。
   - 开关、按钮、状态刷新使用统一时长和缓动。
   - 重复点击当前页面不触发动画。

4. **直播工作流优先**
   - 首页是直播控制中心。
   - AI 助播、关键词回复、场控话术、AI 语音/推流分别承担独立配置任务。
   - 重要状态必须第一时间可见：AI 是否回复、语音是否启用、关键词是否启用、场控是否启用、推流是否就绪。

## 改造范围

### 1. 视觉系统

- 重构 `ui_theme.py` 的颜色、间距、字体、圆角、阴影、动效 token。
- 统一 `MacCard`、`MacButton`、`MacComboBox`、`MacLineEdit`、`SiSwitch` 的视觉语言。
- 减少页面内联 stylesheet，优先复用组件级样式。
- 深色和浅色主题都要保持同一套层级关系。

### 2. 首页

- 将首页定位为“直播控制中心”。
- 保留并强化当前结构：
  - 左侧：直播弹幕。
  - 右侧：快捷操作、自动场控、关键词回复。
  - 底部：音频设备、推流控制。
- 快捷操作按钮改成更像 iOS 控制中心的状态按钮：
  - 图标 + 简短文字。
  - 开启状态有轻微填充色。
  - 关闭状态保持低对比。
- 自动场控、关键词回复卡片进一步压缩文字，只突出状态和关键设置。
- OBS 只保留连接状态点和连接动作，不再占大卡片。

### 3. 侧边栏

- 当前顺序保持：
  - 首页
  - AI 助播
  - 关键词回复
  - 场控话术
  - AI 语音 / 推流
  - OBS 联动
  - 设置
- 当前页高亮改成轻量背景或细线，不使用过重的选中态。
- 图标尺寸、文字对齐、间距统一。

### 4. 配置页

- AI 助播：
  - 保留核心开关和模型设置。
  - 高级提示词、人设、冷却等信息分区折叠。

- 关键词回复：
  - 强调规则编辑效率。
  - 增加清晰的模板状态和当前规则数量。
  - 保存、添加、删除动作统一按钮层级。

- 场控话术：
  - 和关键词回复保持一致的模板/列表编辑模式。
  - 自动发送、随机顺序、随机空格、语音开关聚合在顶部。

- AI 语音 / 推流：
  - 改成流程状态视图：音色 -> 素材 -> 数字人 -> OBS -> 推流。
  - 每一步给出就绪/缺失/运行中状态。

### 5. 动效系统

- 新增统一 motion helper：
  - 页面切换：180-220ms，轻微位移 + opacity。
  - 按钮按下：80-120ms，颜色/亮度/轻微缩放。
  - 卡片状态变化：120-180ms crossfade。
  - 列表新增/删除：短位移 + 淡入/淡出。
- 所有动画使用一致 easing，避免每个页面各自写。
- 禁止重复点击当前页触发页面切换动画。

## 实施阶段

### Phase 1: 视觉 token 和组件统一

- 清理 `ui_theme.py` 中分散的样式常量。
- 统一 Mac 组件视觉。
- 建立按钮状态规范：primary、secondary、pill、destructive、ghost。
- 验收：主要页面组件视觉一致，无明显混搭。

### Phase 2: 首页精修

- 重新打磨首页卡片尺寸、间距、按钮状态。
- 快捷操作改成更高级的状态按钮。
- 音频设备和推流控制改成更清晰的状态面板。
- 验收：首页第一屏信息清楚，按钮不挤压，直播时高频状态一眼可见。

### Phase 3: 侧边栏和页面切换动效

- 优化侧边栏当前页状态。
- 替换大幅上滑为轻量淡入动效。
- 保持当前页重复点击无动画。
- 验收：切页自然，不晃眼，不误触发。

### Phase 4: 配置页降噪

- AI 助播、关键词回复、场控话术、AI 语音/推流逐页整理。
- 把高级选项折叠，把主流程放在上方。
- 验收：每页主任务明确，重复文字减少，操作路径更短。

### Phase 5: 视觉 QA

- 检查 1280x720、1440x900、1920x1080 三档窗口。
- 检查浅色/深色主题。
- 检查按钮文字是否溢出。
- 检查页面切换、开关、状态刷新动画是否卡顿。

## 验收标准

- 首页看起来像一个现代直播控制中心，而不是配置表单集合。
- 左侧导航顺序符合直播工作流。
- 主要按钮状态可读，不需要用户猜当前是开还是关。
- 页面切换和控件动效自然，重复点击当前页不触发动画。
- 常用操作 1-2 次点击完成。
- 高级设置不干扰日常开播操作。

## 首批建议改动

1. 先统一 `ui_theme.py` 和核心组件。
2. 再精修首页，因为视觉提升最明显。
3. 再做统一动效，避免后续每页重复实现。
4. 最后逐页整理配置页。
